# 视障辅助智能导航系统技术架构文档

## 1. 总体架构

### 1.1 架构概览

#### 2.1.1 多引擎ASR架构

系统采用**三级降级策略**，确保在各种网络条件下都能提供语音识别服务：

```
│         └─────────────────────────┘         │                       │
│                      │ 失败                  │ 失败                  │

│         │  优先级2: 系统ASR       │   │  优先级3: 讯飞云端   │       │
```

**实现要点：**
- **VoskAsrEngine**: 离线模型，预加载中文语音包，无网络依赖

- **XunfeiAsrEngine**: 讯飞云端API，最高准确率，需APP_ID/API_KEY/API_SECRET

#### 2.3.2 导航指令友好化转换

系统将专业的地图导航指令转换为视障人士友好的自然语言：

| 原始指令 | 转换后 | 转换规则 |
|---------|--------|---------|
| "沿向前230米" | "直行354步" | 米→步数 (÷0.65) |
| "沿向左150米" | "左转231步然后直行" | 方向简化 |
| "沿向右80米" | "右转123步然后直行" | 方向简化 |

**转换公式：**
```
步数 = round(米数 / 0.65)
方向映射: 沿向前 → 直行, 沿向左 → 左转, 沿向右 → 右转
```

#### 2.4.2 路径聚焦检测策略

为避免检测无关区域的障碍物（如车道上的车辆），系统采用**时钟方向聚焦**策略：

```
                  12点 (正前方)
                      │
              11点 ────┼──── 1点    ← 主要检测区域 (60%画面)
           10点 ────────┼──────── 2点   只关注行走路径
        9点 ─────────────┼─────────── 3点   ← 次要检测区域
                      │
      (忽略)         用户          (忽略)
                      │
                   6点 (后方)
```

**检测区域规则：**
| 时钟方向 | 检测优先级 | 说明 |
|---------|-----------|------|
| 11-1点 | 高 | 正前方盲道区域，重点检测 |
| 9-11点，1-3点 | 中 | 前方偏左/偏右，次要检测 |
| 7-9点，3-5点 | 低 | 两侧区域，仅高优先级障碍物 |
| 5-7点 | 忽略 | 后方区域，不检测 |

### 2.6 TTS优先队列系统

#### 2.6.1 队列架构

为解决多路TTS消息冲突问题，系统基于Redis实现了**优先级队列**：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        TTS 优先队列系统                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                      消息来源 (多路输入)                         │   │
│  ├────────────────┬────────────────┬────────────────┬──────────────┤   │
│  │ 障碍检测 (高)  │   导航指令 (中) │  自动化 (低)   │  其他 (普通)  │   │
│  └────────┬───────┴────────┬───────┴────────┬───────┴──────┬───────┘   │
│           │                │                │              │           │
│           ▼                ▼                ▼              ▼           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                  Redis Sorted Set (ZSET)                       │   │
│  │  Key: tts:queue:{user_id}                                     │   │
│  │  Score: priority (CRITICAL=4, HIGH=3, MEDIUM=2, NORMAL=1)     │   │
│  │  Value: JSON消息 {text, priority, timestamp}                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                    │                                  │
│                                    ▼                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                    Android 定时轮询                              │   │
│  │  GET /api/tts/pull?user_id={user_id}                           │   │
│  │  返回: 按优先级排序的Top 10消息                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                    │                                  │
│                                    ▼                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                      Android TTS 依次播报                        │   │
│  │  - 障碍警报优先播报                                              │   │
│  │  - 导航指令插队播报                                              │   │
│  │  - 其他消息按序播报                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 2.6.2 优先级定义

```java
public enum TtsPriority {
    CRITICAL(4, "紧急障碍物警告，如台阶、楼梯"),
    HIGH(3, "高优先级信息，如低矮障碍物"),
    MEDIUM(2, "中等优先级，如导航指令"),
    NORMAL(1, "普通信息");

    private final int value;
    private final String description;
}
```

#### 2.6.3 队列API接口

**添加TTS消息：**
```http
POST /api/tts/push
Content-Type: application/json

{
  "user_id": "user123",
```

## 6. 技术亮点与创新

### 6.1 多模型协同架构

- **轻量级意图模型** (Qwen2-1.5B): 毫秒级响应，降低延迟
- **专用视觉模型** (Qwen2-VL-7B): 聚焦障碍物检测，路径聚焦策略
- **自动化模型** (AutoGLM-9B): 手机UI理解与操作

### 6.2 智能去重机制

- **时间窗口去重**: 避免短时间重复播报
- **距离变化感知**: 障碍物接近时重新提醒
- **路径聚焦检测**: 只关注行走路径上的障碍物

### 6.3 优先级TTS队列

- **多路消息融合**: 避障、导航、自动化消息统一管理
- **优先级插队**: 紧急消息打断低优先级播报
- **Redis持久化**: 支持分布式部署

### 6.4 多引擎ASR降级

- **离线优先**: Vosk无网络依赖
- **云端兜底**: 讯飞API高准确率
- **平滑切换**: 用户无感知

### 6.5 导航指令友好化

- **绝对→相对方向**: "沿向前" → "直行"
- **米数→步数转换**: 符合盲人行走习惯
- **多模态反馈**: 语音+振动提示
